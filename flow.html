NBA Bet — Automated Flow

End-to-end data pipeline, current state & gaps

1 · Trigger

⚡ How the worker is invoked Not yet automated

The worker is an AWS Lambda function (worker/lambda_function.py). It has no scheduled trigger configured yet — it must be invoked manually (e.g. via the AWS Console or CLI).

Gap: A recurring schedule needs to be created — for example an AWS EventBridge (CloudWatch Events) cron rule firing every ~10 minutes during game hours, or a simpler hourly rule during the playoffs.

3 · Scoring Rules (what Step 5 must implement)

🏀 Game events (play-in, conf finals, finals)
Correct winner2 – 4 pts
Exact point difference4 – 8 pts
Closest point difference3 – 6 pts

"Closest" requires comparing all users' predictions — a cross-bet calculation.

📊 Series events (rounds 1 – 4)
Correct series winner4 – 12 pts
Winner + exact games6 – 16 pts

Points scale by round. Finals are worth 3× first round.
